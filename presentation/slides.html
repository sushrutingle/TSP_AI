SLIDE 03 — PROBLEM DEFINITION

The Travelling Salesman Problem

Problem Statement

"Given n cities, find the shortest route that visits each city exactly once and returns to the starting city."

  • Classified as NP-Hard
  • Brute force: (n-1)!/2 possible tours
  • For 20 cities: 60 quadrillion routes!
  • Requires heuristic / approximate methods

Our System Requirements

  • Accept 2–10 Indian cities as input
  • Compute Haversine pairwise distances
  • Apply Hill Climbing optimization
  • Return cyclic tour (back to start)
  • Visualize on interactive map
  • Display segment-level distance breakdown

SLIDE 04 — MATHEMATICAL MODEL

Objective Function & Haversine Formula

TSP Objective Function

Minimize: f(R) = Σᵢ d(cᵢ, cᵢ₊₁ mod n) Subject to: • each city visited exactly once • route is cyclic (returns to c₀) • d(cᵢ, cⱼ) ≥ 0 ∀ i, j

Where R = route permutation, cᵢ = city at position i, d() = Haversine distance.

Haversine Formula

a = sin²(Δlat/2) + cos(lat₁)·cos(lat₂)·sin²(Δlon/2) c = 2·atan2(√a, √(1−a)) d = 6371 × c km

Accounts for Earth's spherical geometry. Accurate to within 0.5% for most distances.

SLIDE 05 — ALGORITHM

Hill Climbing Algorithm

FUNCTION HillClimbing(cities, restarts):
  best_global ← ∞

  FOR r = 1 TO restarts:
    route ← RandomShuffle(cities)
    dist  ← TotalDistance(route)

    REPEAT:
      improved ← FALSE
      FOR all pairs (i, j):
        nbr ← Swap(route, i, j)
        nd  ← Distance(nbr)
        IF nd < dist:
          route, dist ← nbr, nd
          improved ← TRUE
          BREAK (first-improvement)

    UNTIL NOT improved

    IF dist < best_global:
      best_global ← dist
      best_route  ← route

  RETURN best_route, best_global

How It Works

  • Start: Random permutation of cities
  • Explore: Generate all pairwise swaps
  • Accept: Move to better neighbour
  • Terminate: No improvement found
  • Restart: New random starting point
  • Return: Best across all restarts

Move Types

  • 🔄 City Swap: Exchange positions of two cities
  • ↩ 2-opt: Reverse a sub-segment

SLIDE 11 — ADVANTAGES & LIMITATIONS

Evaluation

✅ Advantages

  • Simple, interpretable algorithm
  • Excellent for 2–15 city problems
  • Memory-efficient (O(n) space)
  • Fast: sub-second for ≤10 cities
  • Random restarts reduce local optima
  • Accurate Haversine distances
  • Interactive, exportable map output
  • No external API required

⚠ Limitations

  • No guarantee of global optimum
  • Greedy — no backtracking
  • Scales poorly beyond ~20 cities
  • Straight-line distances (not road network)
  • No real-time traffic data
  • Quality varies with random seed

🔮 Mitigation

  • Random restarts reduce optima trapping
  • 2-opt gives more diverse neighbourhood
